Areas de GIT

WORKING DIRECTORY

El area de trabajo corresponde a la carpeta donde creamos el respositorio y vamos a crear, editar y/o eliminar los archivos que forman parte del proyecto. Normalmente trabajaremos en esta area con algun editor de codigo o IDE y utilizaremos algun que otro comando circunstancialmente, entre los cuales se encuentran:

  • dir | ls: muestra el contenido de la carpeta actual
  • mkdir <name>: crear una carpeta con el nombre especificado
  • cd <path> abre el directorio seleccionado
  • git init: inicializa un repositorio en la carpeta actual
  • rm -rf .git elimina el repositorio de git

STAGING AREA

El area de control de cambios, tambien llamada INDEX, se utiliza para realizar las capturas de codigo(SNAPSHOT) que luego deberan ser confirmadas si corresponde. Aqui es donde se realiza el seguimiento de los archivos y algunos de los comandos que podemos utilizar son los siguientes:

  • git status: muestra el estado de los archivos que se encuentran en el directorio actual
  • git add <file>: agrega al INDEX los archivos seleccionados
  • git diff: compara el working directory con las capturas del INDEX
  • git restore --staged <file>: Elimina la ultima captura realizada
  • git rm --cached <file>: Elimina del Repositorio y quita del seguimiento un archivo

REPOSITORY

El almacen de cambios corresponde a una carpeta oculta que registra todo en archivos BLOB(Binary Large Object), y que podemos consultar utilizan el historial de confirmaciones.
